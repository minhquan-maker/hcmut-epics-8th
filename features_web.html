Feature Breakdown
Authentication
Phone + password (JWT) or Google OAuth via Firebase. Admin/rescuer registration requires a role verification code.
Profile Settings
Edit name, phone, address, gender, DOB. Upload avatar and change password.
Appearance
Light / Dark / System modes. Language switch between English and Vietnamese. Settings persist across sessions.
News & Alerts
Flood news feed with severity classification — danger (red), warning (orange), info (blue). Push-style notifications.
About Us
System overview, team information, project goals, and mission statement.
AI ChatBot
Fixed bottom-right chat widget. Toggleable open/close. Provides instant support and guidance for all users.
Dashboard
Welcome header with city, safety status toggle (Safe / Danger / Injured), active SOS banner, quick actions, family safety board, and SOS history.
SOS Request
Send emergency requests with auto-detected GPS, description, urgency level (Low / Medium / High / Critical), and up to 5 photos uploaded to Cloudinary.
Live Flood Map
Leaflet-based real-time map showing flood zones, water levels, danger points, SOS markers, and family member locations. Right panel with family check list.
Family Safety
Search and connect family members by phone. Accept/reject invites. Update safety status (Safe / Danger / Injured / Unknown) with health notes. View all on map.
Safety Protocols
Step-by-step guides for before, during, and after a flood. Emergency contact directory — rescue hotlines, hospitals, and police numbers.
Live Tracking
Once a rescuer accepts an SOS, citizens see the rescuer's real-time GPS location on the map via WebSocket — from dispatch to arrival.
Rescue Queue
View all SOS requests sorted by newest first. See full details — name, phone, address, description, urgency, photos, GPS. Requests within 30 min marked as new.
Accept / Complete Mission
Leader and Co-leader can accept SOS from queue. Complete or cancel missions. Members (non-leaders) can view but not operate on missions.
Live Flood Map
Same real-time map as citizens but optimized for rescue — SOS markers, danger zones, team positions. Right panel shows team info and rescue status.
Team Management
Create/join teams. Invite members by phone. Promote Members to Co-leader (Leader only). Kick members, leave team, or disband (Leader only).
Live GPS Sharing
Share real-time GPS location with the citizen during an active mission. All team members see shared positions. Stops automatically on mission complete/cancel.
No Family Features
Rescuers do not have access to family safety features. Focus is entirely on mission coordination and team operations.
Dashboard
Overview stats — Total Users, Rescuers, Pending SOS, Active SOS. Quick SOS summary table with counts by status. Multi-tab navigation.
User Management
Full user list with name, phone, role, created date. Filter by role (Citizens / Rescuers / Admins). Change any user's role via dropdown.
SOS Management
View all SOS requests with status/urgency filters. Assign any SOS to a rescue group. Track mission progress in real time.
Rescue Teams
View all rescue groups — members, leader, status. Per-team detail with member list and mission statistics.
Flood Map Editor
Full map editing capabilities — add, edit, and delete flood zones. Manage danger points and water level markers directly on the Leaflet map.
System Analytics
User growth over time, SOS trends (daily/weekly/monthly), rescue performance metrics (avg response time, fastest, slowest), and role distribution charts.

End-to-End Rescue Flow
1
Flood Occurs

Disaster event detected. Citizens in affected areas begin experiencing danger and need immediate assistance.

2
Citizen Sends SOS

Citizen submits SOS with GPS location, description, urgency level, and up to 5 photos. Status becomes pending.

3
Admin Assigns Group

Admin reviews SOS and assigns it to a rescue team. The team's Leader becomes the assigned responder. Status becomes assigned.

4
Rescuer Tracks & Responds

Leader/Co-leader accepts the mission. WebSocket begins sharing rescuer GPS with the citizen in real time. Status becomes in_progress.

5
Mission Resolved

Rescuer marks the mission complete. Data flows into the analytics dashboard. Status becomes resolved.

API Endpoints
POST
/api/sos
Citizen creates a new SOS request with location, description, urgency, and photo URLs.
Citizen
GET
/api/sos/my
Citizen retrieves all their own SOS requests with current status and assigned rescuer info.
Citizen
GET
/api/sos/all
Retrieve all SOS requests across the system. Filterable by status and urgency level.
Admin / Rescuer
GET
/api/sos/team
Rescuer retrieves all SOS requests assigned to their specific rescue team.
Rescuer
PUT
/api/sos/:id/assign
Admin assigns a specific SOS to a rescue group. The group's Leader becomes the assigned responder.
Admin
PUT
/api/sos/:id/accept
Rescuer (Leader/Co-leader) accepts an SOS from the queue. Starts GPS sharing via WebSocket.
Rescuer
PUT
/api/sos/:id/complete
Rescuer marks the mission as successfully completed. Stops GPS sharing. Status: resolved.
Rescuer
PUT
/api/sos/:id/cancel
Rescuer cancels the mission and returns it to the queue. Stops GPS sharing. Status: cancelled.
